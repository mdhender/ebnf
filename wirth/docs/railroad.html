EBNF Railroad Diagrams

Grammar


    syntax     = { production } .
    production = NONTERMINAL EQ expression TERMINATOR .
    expression = term { OR term } .
    term       = factor { factor } .
    factor     = NONTERMINAL
               | TERMINAL
               | START_GROUP expression END_GROUP
               | START_OPTION expression END_OPTION
               | START_REPETITION expression END_REPETITION .

Combining and simplifying diagrams

Wirth suggests combining and simplifying diagrams before writing parsers.

We can combine syntax and production into a single diagram. We can also combine expression, and term. We can't reduce further because expression and factor are mutually recursive.

syntax

expression

factor

    
Diagram(
    ZeroOrMore(
        Sequence(
            Terminal('NonTerminal'),
            Terminal('='),
            Optional(NonTerminal('expression')),
            Terminal('.'),
        ),
    ),
)

Diagram(
    OneOrMore(
        NonTerminal('factor'),
